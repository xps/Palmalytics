## How to contribute

1. Open an issue first for anything non-trivial, so we can discuss the approach.
2. Fork the repo and create a branch for your change.
3. Keep PRs focused — one logical change per PR is easier to review.
4. Before merging, we'll ask you to confirm in the PR thread that you agree to the terms above.
